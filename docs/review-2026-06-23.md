## 1. 🔴 CRITICAL — 즉시 (오늘)
**`.env` / `.env.local`에 라이브 운영 시크릿 평문 노출**
- Neon DB 비밀번호, `AUTH_SECRET`(JWT 서명키), Google OAuth 시크릿, Vercel OIDC 토큰.
- git에는 없음(확인됨), 하지만 디스크에 평문 + **이 디렉터리를 AI 에이전트/툴에 계속 넘기고 있음** → 사실상 노출로 간주.
- `AUTH_SECRET` 유출 시 **임의 유저 세션 위조 가능 → level≥10 위조해 관리자 탈취**.
- **조치(순서대로 회전):** ① Neon 롤 비번 재발급 → ② `AUTH_SECRET` 재생성(`openssl rand -base64 32`, Vercel env 반영, 전 세션 무효화) → ③ Google OAuth 시크릿 재발급 → ④ Vercel OIDC 재발급.
- 이후 로컬은 `vercel env pull`로만 받기.

## 2. 🟠 HIGH — 이번 주
**(A) 크론 일 1회 vs 캐시 TTL 1~5분 불일치** *(3개 에이전트가 동시 지적, 최대 임팩트)*
- `vercel.json` = `0 0 * * *`(하루 1회). 그런데 `cache.ts`/`ticker.ts` 주석은 "크론이 60초마다 미리 갱신"을 전제.
- 결과: 거의 모든 첫 방문자가 TTL 만료 후 **외부 API 동기 호출(최대 8s)을 요청 경로에서 직접** 부담. `kimpSnapshot`(5분 의도)/`marketSnapshot`(15분 의도)이 사실상 하루 1회만 기록 → 스냅샷 히스토리 손상.
- **조치:** 플랜이 허용하면 `*/2 * * * *`(또는 `*/5`)로 변경. Hobby 플랜이라 일 1회 제약이면 → 주석을 현실에 맞게 수정하고 inline-refresh를 정식 경로로 인정. **플랜 확인 후 결정 필요.**

**(B) `revalidatePath` 보드 경로 오류** (`lib/actions.ts:119,194`)
- `createComment`/`votePost`가 항상 `/free/${postId}` 무효화. 그러나 글은 `/analysis/[id]` 등 다른 보드에도 존재 → **analysis/forum 글의 댓글·투표가 stale**.
- **조치:** 보드 slug를 받아 실제 `${basePath}/${postId}` 무효화. `createPost`의 basePath 로직 재사용.

**(C) Rate limiter fail-open + XFF 스푸핑 가능** (`lib/ratelimit.ts:31-41`)
- `x-forwarded-for` 첫 값을 그대로 키로 사용 → 공격자가 헤더 바꿔가며 무제한 로그인/가입 시도(브루트포스). DB 에러 시 `catch{return true}`로 **무력화**.
- **조치:** Vercel 신뢰 헤더(`x-vercel-forwarded-for`) 사용. `login:`/`register:` 접두사는 **fail-closed**(에러 시 거부)로.

**(D) listings 스크래퍼 SSRF — TOCTOU/DNS 리바인딩** (`lib/listings.ts`)
- `isSafePublicUrl`가 DNS를 1회 검사 후 `fetch`가 재해석 → 검사 통과 후 사설 IP로 리바인딩 가능. `http:` 다운그레이드 허용.
- (방어 자체는 우수: redirect manual, 사설대역 차단 등.) 약점만 보완.
- **조치(가장 견고):** deny-list 대신 **거래소 도메인 allow-list**(binance/bithumb/upbit/bybit/coinbase/robinhood) + `https:`만 허용. allow-list면 리바인딩 무력화.

## 3. 🟡 MEDIUM — 이번 달
- **fxCache 무한 stale**(`ticker.ts:55-57`): FX 실패 시 나이 체크 없이 `source:"cached"`로 옛 환율 신뢰 처리 → 김프 수치 왜곡. **6h 상한 두고 초과 시 `fallback`으로 강등.**
- **로그인 이메일 정규화 누락**(`auth.ts:31`): 가입은 `trim().toLowerCase()`, 로그인은 원문 → 대문자 입력 시 자동로그인/재로그인 실패. **로그인도 동일 정규화.**
- **cron health 거짓 양성**(`cron/market/route.ts:34`): 워머들이 fallback 페이로드 반환 → 전부 실패해도 `ok:true/200`. **stale `updatedAt`/빈 페이로드를 실패로 간주.**
- **`postId` 검증 없음**(`actions.ts:105,168`): 클라 `postId` 그대로 신뢰. FK로 막히지만 raw 500 노출 + analysis 보드 정책 우회 가능. **존재/보드 정책 선검증.**
- **`openRandomBox` 잔액 오표시**(`actions.ts:320`): 트랜잭션 전 `me.points`로 `-BOX_COST` 계산 → 동시성 시 표시 잔액 오류. **트랜잭션 반환값 사용.**
- **방문자 카운터 무방비**(`api/track`): 인증·레이트리밋 없는 `increment` → `curl` 루프로 임의 부풀리기. **IP 레이트리밋/세션 쿠키 dedupe.**
- **postcss < 8.5.10 CVE**(transitive via next): moderate. `--force` 금지(next 9 다운그레이드됨). **`overrides`로 패치 버전 핀 or next 패치 대기.**
- **보안 헤더/CSP 부재**(`next.config.ts` 비어있음): CSP, frame-ancestors, nosniff, HSTS 없음. **`headers()` 추가.**

## 4. 🟢 편의성 / UX 개선 (제품)
**Quick wins (S):**
- 클라 fetch 컴포넌트(`TickerTable`,`TickerBar`,`BubbleMap`) **에러 상태 전무** → 실패 시 무한 로딩 멈춤. catch→"불러오기 실패·재시도" 추가.
- 글쓰기 폼(`free/write`,`analysis/write`) **제출 pending/disabled 없음** → 중복 제출·먹통 클릭. `useTransition` 적용.
- `CryptoCalendar` 필터 / `BubbleMap` 기간 → **localStorage 영속화**(현재 새로고침 시 초기화).
- `BubbleMap`,`LiveSparkline` 백그라운드 탭에서도 폴링 → `document.hidden` 체크(`TickerTable` 패턴 복사).
- 스켈레톤 로더로 텍스트 로딩 대체(체감 성능).

**Medium (M):**
- `MarketCards.tsx:42` 매 렌더마다 `take:336`(14일치) 과다 페치 → 스파크라인 필요량만.
- `PostView`에 링크 복사/공유 버튼.
- BubbleMap 모바일 터치 미지원(`onMouseEnter`만) → 탭 인터랙션 추가.

**Larger bets (L):**
- **LangToggle는 현재 장식용**(EN 커버리지 0%, 모든 문자열 하드코딩 KO). 실제 i18n(next-intl 등) 도입 여부 결정.
- 신규 상장/가격 알림(웹푸시), 커뮤니티 보드 검색/정렬(스키마 인덱스는 이미 존재).
