# 코인판(coinpan.com) 사이트 구조 및 기능 분석

> 작성일: 2026-06-10
> 용도: 사내 교육용 / 동일 유형 사이트(크립토 시장 분석 콘텐츠 포함) 구축 참고 자료
> 분석 대상: https://coinpan.com (메인), https://coinpan.com/free (자유게시판), 게시글 상세 페이지

---

## 1. 사이트 개요

코인판은 국내 최대급 **암호화폐 커뮤니티 사이트**다. 핵심 가치는 두 가지 축의 결합이다.

1. **실시간 시세 데이터** — 국내외 거래소 시세, 김치프리미엄, 거래량을 항상 노출
2. **커뮤니티 게시판** — 자유게시판 중심의 고회전 익명성 커뮤니티 + 코인별 포럼

즉, "시세를 보러 왔다가 글을 읽고, 글을 읽으러 왔다가 시세를 본다"는 순환 구조로 체류 시간과 재방문을 만들어내는 모델이다. 우리가 만들 사이트도 이 **데이터(시세/분석) + 커뮤니티(토론)** 이중 구조를 기본 골격으로 삼으면 된다.

## 2. 기술 플랫폼

| 항목 | 내용 |
|------|------|
| CMS | **XpressEngine(XE) 또는 Rhymix** (PHP 기반 국산 CMS) |
| URL 패턴 | `/index.php?mid=free&act=...` (모듈 방식), 짧은 주소 `/free/470145013` (게시판ID/문서ID) |
| 게시판 스킨 | `/modules/board/skins/WhiteBoard_dark/` — "WhiteBoard_dark" 커스텀 스킨 |
| 구조 | 모듈형: 게시판(board) 모듈, 포인트 모듈, 회원 모듈이 분리되어 조합됨 |
| 테마 | 야간모드(다크모드) 토글 지원 |
| 다국어 | 한국어 / English / 日本語 / 中文 선택 가능 |

**교육 포인트:** XE/Rhymix는 `mid`(모듈 인스턴스 ID) 하나로 게시판을 무한 증설하는 구조다. 코인판의 수십 개 게시판은 전부 같은 board 모듈의 인스턴스이며, 스킨과 권한 설정만 다르다. 현대적 스택(Next.js + DB)으로 재구현한다면 "board 테이블 + board_id를 가진 post 테이블" 구조가 정확히 대응된다.

## 3. 정보 구조(IA) — 전체 메뉴 트리

```
코인판 (메인)
├── 코인판
│   ├── 코인정보          ← 정보성 게시판
│   ├── 코인뉴스          ← 뉴스 큐레이션
│   ├── 차트분석          ← ★ 시장 분석 콘텐츠가 들어가는 곳
│   ├── 채굴정보 / 채굴게시판 / 채굴기장터
│   ├── 에어드랍
│   └── 출석체크          ← 리텐션 장치 (포인트 지급)
├── 선물/마진
│   ├── 선물/마진 정보
│   └── 선물/마진 게시판
├── 커뮤니티
│   ├── 자유게시판        ← 트래픽 핵심 (고회전)
│   ├── 정치/시사
│   ├── NFT 게시판
│   ├── 손익인증 게시판    ← 수익/손실 스크린샷 인증 문화
│   ├── 질문/답변
│   ├── 유머/감동
│   ├── HOT게시물 / 추천게시물  ← 추천수 기반 자동 집계 게시판
│   └── 가입인사
├── P2E 게임 (정보 / 게시판)
├── 포럼               ← 코인별 전용 게시판
│   ├── 전체글 / HOT게시물
│   └── 코인별: 비트코인, 이더리움, 리플, 솔라나, 에이다, 도지코인,
│       루나, 이오스, 샌드박스, 위믹스, 디센트럴랜드, 카이아,
│       비트코인캐시, 이캐시, BSV, 이더리움클래식, 라이트코인,
│       스텔라루멘, 퀀텀, 트론, 메이저코인, 알트코인
├── 갤러리 (폰카 / 자유 / 연예인)
├── 거래소/차트
│   ├── 국내거래소 / 해외거래소 / NFT거래소  ← 제휴(레퍼럴) 링크
│   └── 차트/시세
└── 공지
```

**구조적 특징 3가지:**

1. **콘텐츠 성격별 분리**: 정보(코인정보·뉴스·차트분석) vs 커뮤니티(자유·유머) vs 거래(거래소 링크)가 명확히 나뉜다.
2. **자동 집계 게시판**: HOT게시물·추천게시물은 글을 쓰는 곳이 아니라 추천수/조회수 임계값을 넘은 글이 자동으로 모이는 가상 게시판이다.
3. **코인별 포럼**: 코인 하나당 게시판 하나. 종목 토론을 자유게시판에서 분리해 검색성과 깊이를 확보한다.

## 5. 핵심 기능 ① — 실시간 시세 위젯

코인판의 차별화 요소이자, 우리 사이트에서 "시장 분석"과 직결되는 부분.

| 요소 | 상세 |
|------|------|
| 대상 거래소 | 국내: 빗썸, 업비트, 코인원, 코빗 / 해외: 바이낸스, 비트파이넥스, 비트플라이어 |
| 표시 코인 | BTC, ETH, ETC, XRP, TRX, BCH, ADA, SOL, DOGE 등 주요 코인 |
| 표시 항목 | 실시간 가격(KRW/USD), 24시간 변동률(상승=빨강/하락=파랑, 국내 관례), 거래량 |
| **김치프리미엄** | 국내가 vs 해외가(환율 적용) 괴리율을 % 표시 — 코인판의 시그니처 기능. 푸터에도 "김프" 전용 외부 링크 존재 |
| 시장 요약 | 전체 시장 24H 볼륨, **비트코인 도미넌스(점유율)** 상단 고정 표시 |

**구현 관점 (우리 사이트 적용 시):**
- 국내 시세: 업비트/빗썸 공개 API (WebSocket 실시간 또는 REST 폴링)
- 해외 시세: 바이낸스 API
- 환율: 달러/원 환율 API → 김프 = (국내가격 / (해외가격 × 환율) − 1) × 100
- 도미넌스/총 시총: CoinGecko 또는 CoinMarketCap API

## 6. 핵심 기능 ② — 게시판 시스템

### 6.1 목록 페이지

| 컬럼 | 형식 |
|------|------|
| No | 글 번호 (공지는 "공지", 광고는 "AD" 라벨) |
| 제목 | 링크 + 댓글수 `[3]` + 새글/이미지/첨부 아이콘 |
| 닉네임 | 레벨 아이콘 + 닉네임 (클릭 시 팝업 메뉴) |
| 등록일 | 당일은 `21:39` 시각만, 이전 글은 날짜 — **상대적 시간 표기** |
| 조회수 | 숫자 |
| 추천 | `추천 - 비추천` (예: `2 - 0`) |

- **정렬**: 등록일/조회/추천 컬럼 헤더 클릭으로 정렬 가능
- **검색**: 제목+내용 / 제목 / 내용 / 댓글 / 닉네임 / 태그 — 6가지 범위 선택
- **페이지네이션**: 1~10 + 이전/다음 블록 (무한스크롤 아님)
- **상단 고정**: 공지글 + 광고(AD)글이 목록 최상단 고정

### 6.2 게시글 상세 페이지

| 영역 | 기능 |
|------|------|
| 상단 메타 | 제목, 프로필 이미지+닉네임, 추천/비추천 수, 댓글 수, 조회수, 작성일시, **작성자 IP 일부 표시** |
| 본문 | 이미지 첨부, 첨부파일(로그인 후 다운로드) |
| 하단 버튼 | 추천/비추천, 스크랩, 인쇄, 신고, 목록, 위로 |
| 태그 | 글에 태그 부여 → 태그 검색 가능 |
| 하단 | 목록으로 복귀 + 같은 게시판 목록 일부 노출 (연속 소비 유도) |

### 6.3 댓글 시스템

- 로그인 회원만 작성 가능
- 댓글 추천/비추천
- **핫코멘트**: 추천 5개 이상 받은 댓글을 댓글 영역 최상단에 별도 노출 ★
- 댓글 새로고침 버튼 (페이지 전체 리로드 없이 갱신)
- **비회원 열람 지연**: "비회원은 N분 후 확인 가능" — 댓글 즉시 열람을 회원 혜택으로 만들어 가입을 유도하는 장치 ★

## 9. 우리 사이트(coinmap) 설계에 주는 시사점

목표: **이 유형의 사이트 + 정확한 크립토 시장 분석 콘텐츠**

### 9.1 그대로 가져올 골격

- 2단 레이아웃 (콘텐츠 + 시세/인기글 사이드바)
- 상단 실시간 시세 티커 + 김치프리미엄
- 게시판 공통 엔진 (하나의 board 스키마로 다수 게시판 운영)
- 추천 기반 HOT 자동 집계, 핫코멘트
- 다크모드

### 9.2 코인판 대비 강화할 부분 (차별화 = 시장 분석)

코인판의 "차트분석" 게시판은 **유저 자유 기고**라서 품질 편차가 크다. 우리는 여기를 구조화한다:

1. **공식 시장 분석 섹션** — 에디터/애널리스트가 작성하는 정기 시장 브리핑 (일간/주간), 일반 게시판과 분리된 전용 레이아웃 (차트 임베드, 데이터 테이블)
2. **데이터 기반 대시보드** — 김프 추이 차트, 도미넌스 추이, 공포·탐욕 지수, 펀딩비 등을 글이 아닌 **위젯**으로 상시 제공
3. **분석 글 신뢰도 장치** — 작성 시점 가격 자동 기록, 이후 실제 가격과 비교 표시 (예측 검증), 출처 표기 필수화
4. **코인별 포럼 + 코인별 데이터 결합** — 코인 포럼 상단에 해당 코인의 시세/차트/온체인 요약 고정

### 9.3 권장 기술 스택 (현대화)

| 영역 | 코인판 (현재) | 권장 |
|------|--------------|------|
| 프레임워크 | PHP XE/Rhymix | Next.js (App Router) + TypeScript |
| DB | MySQL | PostgreSQL (Prisma/Drizzle) |
| 실시간 시세 | 폴링/위젯 | 업비트·바이낸스 WebSocket + 서버 캐시(Redis) |
| 차트 | 이미지 첨부 | TradingView 위젯 임베드 또는 lightweight-charts |
| 외부 데이터 | — | CoinGecko API (시총/도미넌스), Alternative.me (공포탐욕지수) |
| 인증 | 자체 회원 | NextAuth/Supabase Auth (이메일 + OAuth) |

### 9.4 단계별 구축 제안

1. **Phase 1**: 시세 티커 + 김프 + 단일 자유게시판 + 회원 (MVP)
2. **Phase 2**: 시장 분석 섹션(공식 콘텐츠) + 데이터 대시보드
3. **Phase 3**: 코인별 포럼, 포인트/레벨, HOT 집계, 출석체크
4. **Phase 4**: 광고/제휴 슬롯, 다국어, 모바일 최적화

## 부록 A. 게시판 DB 스키마 초안 (코인판 구조의 현대적 번역)

```
boards      (id, slug, name, type[community|info|forum], coin_symbol?, sort_order)

             upvotes, downvotes, comment_count, is_notice, is_ad,
             price_at_post?, created_at)        ← price_at_post: 분석글 검증용
comments    (id, post_id, user_id, parent_id?, content, upvotes, downvotes, created_at)
```
